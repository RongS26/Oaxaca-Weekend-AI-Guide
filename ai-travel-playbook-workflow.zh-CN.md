# AI 旅行手册工作流

[English](ai-travel-playbook-workflow.md) | [中文](ai-travel-playbook-workflow.zh-CN.md)

## 目标

用 AI 生成一种能提升文化理解、降低现场决策摩擦，并且能结合个人背景持续修正的旅行 artifact。

最终产物不应该只是 itinerary，而应该是一份可以使用、可以被审查、也可以复用到未来旅行的 guide 和 playbook。

## 输入 Brief

```text
目的地：
旅行日期 / 时长：
出发区域：
当前生活背景：
体力水平：
旅行风格：
必选兴趣：
限制条件：
语言需求：
输出格式：
```

## 工作流

### 1. 低假设生成初版

先给 AI 一个真实旅行 brief：

- 目的地和日期
- 可用时间
- 已知旅行偏好
- 希望的输出格式
- 要求详细行程
- 要求涉及买票、预约或 tour 的地方附链接

目标是先拿到一个足够好的第一版 artifact，而不是一开始就过度限制它。

### 2. 个人上下文审查

拿到第一版后，用个人背景重新审查：

- 哪些体验和我现在居住地附近已经能接触到的东西重复？
- 哪些内容文化上有意思，但价格不值得？
- 哪些付费 tour 可以变成自我探索？
- 哪些推荐看起来像泛泛建议，而不是基于我的偏好？

### 3. 路线、地图和链接要求

继续要求 AI 给出：

- 地图标记点
- 路线顺序
- 实用链接
- 购票 / 预约链接
- Plan A / Plan B
- 付费 tour 和自我探索之间的取舍

### 4. Weekend Playbook

把修正后的 guide 继续转成 playbook：

- 时间块
- 决策点
- Plan A / Plan B
- 体力消耗预估
- 给未来自己的备注

### 5. 视觉和格式迭代

当某一版的视觉效果已经好时，要明确告诉 AI 保留好的部分：

```text
请沿用第一版里我认可的视觉方向和排版优点。
只改弱的部分，不要推翻整体风格。
请保持清晰层级、路线/地图证据和易读的双语格式。
```

### 6. Post-Trip Review

旅行结束后记录：

- 哪些判断准确
- 哪些信息错误
- 哪些内容真的帮助了文化理解
- 哪些 prompt pattern 值得复用

## 可复用 Prompt

```text
请帮我为 [目的地] 和 [日期] 做一份 bilingual travel guide / playbook。

请结合我的旅行时间、当前生活背景和你已经知道的我的旅行偏好。
需要包含详细行程；涉及买票、预约或 tour 的地方，请附链接。

然后请基于我的个人背景重新审查这份方案：
哪些体验和我现在已经能接触到的地方重复？
哪些付费 tour 性价比不高？
哪些可以改成自我探索？

最后请把它整理成一份可执行 playbook：
包含路线逻辑、地图点位、决策点、备选方案和清晰易读的双语格式。
```
