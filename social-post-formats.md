# Social Post Formats

This file turns the Oaxaca AI guide project into reusable social content formats.

It is based on public platform patterns:

- Xiaohongshu / Rednote: practical life guide, searchable notes, visual-first "what I did / what worked" format.
- X: concise build-in-public thread, process-first, strong before/after.
- Jike: reflective practice note, more conversational and process-oriented.

The goal is not to copy any specific post. The goal is to extract a format that fits this project.

## Reference Notes

The format choices below combine three inputs:

- Public platform structure: Xiaohongshu / Rednote is strong for searchable lifestyle notes, travel recommendations, and practical "what worked" posts.
- AI travel planning research: good AI travel output depends on personalization, real-world constraints, route coherence, and adaptability when plans change.
- Local skill guidance: the social-card workflow should treat screenshots, PDF previews, map snippets, and prompt snippets as the evidence layer, not as decoration.

For this project, the strongest public-facing story is:

```text
I did not ask AI for a generic itinerary.
I used AI to turn travel intent into a reusable bilingual guide-making workflow.
```

The reusable angle is more important than the destination itself.

## Platform Observations

### Xiaohongshu / Rednote

Best fit:

- "我用 AI 做了一个周末旅行手册"
- before/after
- step-by-step screenshots or PDF previews
- practical checklist
- soft personal tone

Recommended structure:

```text
Title: 我用 AI 做了一份 Oaxaca 周末双语旅行手册

1. 为什么不是普通攻略
2. 我给 AI 的关键要求
3. 从 Guide 到 Playbook 的 4 次迭代
4. 最终成品长什么样
5. 这套方法可以复用到任何城市
6. Prompt 模板
```

Carousel page plan:

| Page | Role |
|---|---|
| 1 | Hook: AI 做旅行攻略，重点不是省时间 |
| 2 | Problem: 普通攻略太像景点清单 |
| 3 | Method: 先文化逻辑，再路线逻辑 |
| 4 | Iteration: Guide -> Self-guided -> Playbook V1 -> V2 |
| 5 | Artifact: PDF previews / route example |
| 6 | Template: reusable prompt |
| 7 | Reflection: 生活类 AI 资产该怎么沉淀 |

Post body draft:

```text
这次做 Oaxaca 周末攻略，我最满意的不是 AI 给了我很多地点，而是它帮我把“想去旅行”变成了一个可以实际使用的双语 playbook。

我发现关键不是问：帮我做一个 itinerary。
而是问：先解释这个城市的文化和街区逻辑，再把它变成一条我能自己走下来的路线。

最后产物从 bilingual guide 迭代到 self-guided guide，再到 weekend playbook v1/v2。

我会把这种生活类 AI 实践沉淀为：
旅行意图 -> 文化理解 -> 路线逻辑 -> 可执行手册 -> 复盘模板

不是存一堆景点，而是存一个以后可以复用的 AI 协作方法。
```

## X Thread Format

Best fit:

- build-in-public
- concise process
- artifact link
- reusable prompt

Thread draft:

```text
1/ I used AI to turn a weekend Oaxaca trip into a bilingual self-guided guide and playbook.

The useful part was not "AI gave me places to visit."
The useful part was the workflow.

2/ Bad prompt:
"Give me an Oaxaca itinerary."

Better prompt:
"Explain the cultural and neighborhood logic first, then turn it into a self-guided route."

3/ The iteration path:
Travel intent
-> bilingual guide
-> revised self-guided guide
-> weekend playbook v1
-> weekend playbook v2

4/ What made it work:
- context before itinerary
- route logic before schedule
- bilingual output
- decision points
- backups
- offline usability

5/ My reusable prompt:
[insert prompt]

6/ Small lesson:
For life workflows, don't store one-off facts.
Store the pattern that changes future behavior.
```

## Jike Format

Best fit:

- reflective note
- what I learned from using AI
- not overly polished

Draft:

```text
今天把一个很生活的小项目沉淀下来了：用 AI 做 Oaxaca 周末双语旅行手册。

一开始只是想要攻略，后来发现真正有价值的是过程：

普通攻略回答“去哪儿”。
这次 AI 协作更像是在回答“我为什么要这样理解这个地方，以及我到了现场怎么做决定”。

最后从 bilingual guide 到 self-guided guide，再到 playbook v1/v2。

这个过程让我更明确一条规则：
生活类信息不是不值得存，而是不能存表象。

不存“这个景点几点开”。
存“我如何让 AI 帮我把文化理解、路线逻辑和行动手册连接起来”。
```

## References

- Xiaohongshu / Rednote is commonly used for life-guide, search-oriented notes and travel/product recommendations; its note and search behavior makes "practical artifact + reusable method" a better fit than a pure reflection.
- X is structurally suited to short process threads and build-in-public notes: start with the before/after, show the workflow, then link the artifact.
- Jike-style practice posts usually work better when written as concise process reflection rather than polished marketing copy.
- AI travel planning research emphasizes personalization, real-world constraints, feasibility, and adaptability; this project is a small personal version of the same direction.
